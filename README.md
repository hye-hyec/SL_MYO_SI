# SL MYO SI

하루를 시작하기 전, 딱 한 곳에서 챙겨보는 모닝 대시보드.

## 프로젝트 개요

**SL MYO SI**는 바쁜 아침에 여러 탭을 열어가며 정보를 수집하는 번거로움을 없애기 위한 웹 서비스입니다.  
날씨, 뉴스, 오늘의 생각거리 등을 한 화면에서 제공하여 하루를 의미 있게 시작할 수 있도록 돕습니다.

## 핵심 목표

- **정보 집약**: 아침에 필요한 정보를 한 페이지에서 확인
- **생각 유도**: 단순 정보 소비를 넘어 하루를 계획하고 성찰할 수 있는 콘텐츠 제공
- **빠른 소비**: 출근·등교 전 짧은 시간 안에 훑어볼 수 있는 가벼운 UX

## 주요 콘텐츠 (기획)

### 날씨

- 위치 기반으로 **2개 이상의 기상 데이터 소스**를 병렬 수집해 교차 검증
- 단순 기온 표시가 아닌, 체감 온도·강수 확률·미세먼지 등 외출 판단에 필요한 정보 제공
- 예: 기상청 API + OpenWeatherMap 조합

### 뉴스 — 4개 카테고리

전날 또는 이번 주 이슈화된 기사를 큐레이션. **이미 노출된 기사는 제외**하고 항상 새로운 콘텐츠를 제공.  
뻔한 헤드라인보다 사람들이 실제로 관심 가지거나 관심 가질 만한 주제를 우선 선별.

| 카테고리 | 포함 주제 |
|---|---|
| 경제 | 주식·기업 동향, 부동산, 은행·금리, 환율, 금값, 대출 |
| IT·과학 | 반도체, 자동차·모빌리티, 물리·화학·지구과학, 양자컴퓨팅, 생물·의학, 에너지(전기·가스) |
| 사회 | 교육, 정치, 법률·판례, 정부 정책, 주요 사건·사고 |
| 문화 | OTT·드라마·영화, 유튜브·크리에이터, 연예·스포츠, 요리·건강 트렌드 |

**큐레이션 원칙**
- 자극적인 클릭베이트 기사는 배제, 맥락 있는 이슈 위주로 선별
- 카테고리당 노출 기사 수 제한 (예: 3~5건) → 핵심만 빠르게
- 사용자가 본 기사·카테고리 이력을 저장해 중복 노출 방지
- 정기적(매일 아침 또는 일정 주기)으로 콘텐츠 갱신

### 문학·철학 글귀

단순한 명언 한 줄이 아닌, **읽고 생각할 수 있는 분량(500자 이상)**의 텍스트를 매일 제공.

**수록 유형**

| 유형 | 예시 |
|---|---|
| 책 발췌 | 소설·자기개발서·시집·과학도서의 인상적인 문단 |
| 인물 기록 | 유명인의 인터뷰, 연설, 대화록, 사고방식 |
| 사자성어 | 고사성어의 유래 이야기 + 현대적 해석 |

**생성 방식**
- AI가 매일 유형을 선택해 콘텐츠를 생성 (책 발췌·인물 기록·사자성어 중 하나)
- 모든 텍스트는 **한국어**로만 출력 (원문이 외국어·한문이더라도 번역 후 제공)
- 생성 이력을 저장해 같은 주제·인물·사자성어의 재출력 방지

**표기 원칙**
- 책 발췌 → 책 제목, 저자명 반드시 포함
- 인물 기록 → 인물 소개(누구인지, 어떤 배경의 발언인지) 포함
- 사자성어 → 한자 원문, 뜻풀이, 유래 고사 순으로 구성
- 하루 1편, 매일 교체

## 타겟 사용자

- **학생 (10~20대)**: 오늘 해야 할 것들을 정리하고 동기부여가 필요한 사람
- **젊은 직장인 (20~30대)**: 출근 전 세상 돌아가는 흐름을 빠르게 파악하고 싶은 사람
- **커플·부부**: 함께 하루를 시작하는 루틴으로 활용

## 레이아웃

반응형 단일 페이지. 섹션은 위에서 아래로 순서대로 배치.

```
┌─────────────────────────────────────────┐
│              날씨 (전체폭)                │
├─────────────────────────────────────────┤
│  뉴스 — 경제    │    뉴스 — IT·과학      │
├─────────────────────────────────────────┤
│  뉴스 — 사회    │    뉴스 — 문화         │
├─────────────────────────────────────────┤
│         문학·철학 글귀 (전체폭)            │
└─────────────────────────────────────────┘
```

- **PC**: 뉴스 4개 카테고리는 2열 그리드, 나머지 섹션은 전체폭
- **모바일**: 모든 섹션 1열로 순서대로 스택

## 기술 스택

| 역할 | 선택 | 이유 |
|---|---|---|
| 앱 프레임워크 | Streamlit (Python) | 코드 최소화, UI 자동 구성 |
| AI 콘텐츠 생성 | Claude API (Anthropic) | 글귀 주제 생성 |
| 날씨 | 기상청 API + OpenWeatherMap | 2개 소스 교차 검증 |
| 뉴스 | 네이버 뉴스 RSS | 카테고리별 최신 기사 수집 |
| 이력 저장 | JSON 파일 | 노출된 기사·콘텐츠 중복 방지 |
| 배포 | Streamlit Community Cloud | 무료, GitHub 연동으로 간단 배포 |

## 개발 원칙

1. **모르면 멈추고 물어본다** — 불명확하거나 해석이 갈리는 부분은 가정하지 않고 사용자에게 선택지를 제시한 뒤 확인 후 진행
2. **최소한의 코드** — 추측성 코드, 미리 대비하는 코드 금지. 지금 필요한 것만 구현
3. **수정은 꼭 필요한 곳만** — 기능 추가·수정 시 관련 없는 코드는 건드리지 않음
4. **명확한 성공 기준** — 구현 전 완료 기준을 먼저 정의하고, 기준 충족까지 테스트 반복
